## 12. 验收标准

- [ ] 玩家能在场景中获得题目需要的全部数据。
- [ ] 地图轮廓、尺度、高差、生态和动线与相邻关卡有至少三项实质差异。
- [ ] 俯视布局、尺寸、地标、路径、碰撞区和题目道具位置完整。
- [ ] 地形、工具、植被、动物和奖励均写明外观、数量、位置、行为与作用。
- [ ] 建筑、营地、码头和玩法工具的摆放符合环境与功能逻辑。
- [ ] 知识点已在 `design/curriculum_map.md` 标为 `approved`。
- [ ] 题源、版权状态和难度评分完整。
- [ ] 已使用不同方法完成两次验算。
- [ ] 题目不能脱离当前场景原样成立。
- [ ] 正确答案和场景对象数量完全一致。
- [ ] 每类错误答案都有针对性反馈并允许重试。
- [ ] 正确后发生可观察的世界变化。
- [ ] 正确答案只解锁行动，玩家必须亲自完成与答案数量一致的操作。
- [ ] 每次语言提示都有对象、动画、声音、收藏槽或道路状态作为可见证据。
- [ ] 获得物先在世界中出现，再进入持续可见的收藏/背包展示。
- [ ] 玩家能够操作角色验证数学结果。
- [ ] 玩家能返回已解锁的前置区域，且完成状态不会丢失。
- [ ] 边界、碰撞和重置行为正确。
- [ ] 可以明确进入下一关。
- [ ] 已加入确定性的自动流程检查。
